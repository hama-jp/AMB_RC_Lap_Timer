AMB_RC_Lap_Timer
================

AMB P3 デコーダーに対応した、個人練習用のラップタイマー Web アプリです。
このフォルダの中身を USB メモリに入れて持ち運ぶだけで使えます。
ネット接続は不要です(LAN 内専用)。


■ 必要なもの
-------------

- Windows 8.1 / 10 / 11 の PC(管理者権限不要)
- AMB P3 デコーダーがつながっている LAN(LAN ケーブル または同じ WiFi)
- ラップタイムを表示するスマホ / タブレット(同じ LAN に接続)


■ 起動手順
-----------

1. このフォルダを USB メモリ直下など、書き込み可能な場所に置きます。
   (デスクトップでも OK)

2. config.json をメモ帳で開き、AMB の IP アドレスとポートを設定します。
   初回はこのフォルダ内の config.example.json が自動でコピーされます。
       "upstream": {
         "host": "192.168.1.21",   ← AMB の IP に書き換える
         "port": 5403
       }

3. 初回だけ、Windows Defender Firewall に gateway.exe の通信を許可します。
   同梱の setup-firewall.bat を **ダブルクリック** するだけで完了します:

       setup-firewall.bat
        → UAC ダイアログ「このアプリがデバイスに変更を加えることを許可しますか?」
           で「はい」を押す
        → Firewall ルール追加 → 「OK: ルール追加完了」と表示されたら
           何かキーを押してウィンドウを閉じる

   - .bat は同じフォルダの gateway.exe をパス自動検出で許可します。
     ドライブレター(C:, F:, D: 等)が変わっても編集不要です。
   - 別の PC に持ち込んだら、その PC で 1 回だけ再ダブルクリックが
     必要です(Defender Firewall は絶対パスでルールを管理するため)。
   - 何度実行しても同じ DisplayName のルールを置き換えるだけなので
     副作用はありません。
   - 後で削除したいとき(管理者 PowerShell で):
       Remove-NetFirewallRule -DisplayName "AMB RC Lap Timer (gateway.exe)"

   別パターン:
   - PC のネットワークが「プライベート」の場合、初回起動時に
     Defender Firewall ダイアログが自動で出ます。その場合は
     「プライベートネットワーク」にチェックを入れて「アクセスを
     許可する」を押せば、setup-firewall.bat の実行は不要です。
   - .bat を使わずに自分でルールを足したい場合は、管理者 PowerShell で:

       New-NetFirewallRule -DisplayName "AMB RC Lap Timer (gateway.exe)" `
           -Direction Inbound -Action Allow -Profile Any `
           -Program "<このフォルダ>\gateway.exe" -Protocol TCP

4. gateway.exe をダブルクリックして起動します。
   初回だけ Microsoft Defender SmartScreen の警告が出ます:
       「詳細情報」 → 「実行」 を押してください。
   2 回目以降は警告は出ません。

5. 黒いウィンドウ(コンソール)が開いたら、ログに次のような行が出るのを
   待ちます:
       INFO  source: real (TCP)  addr=192.168.1.21:5403
       INFO  upstream connected  addr=192.168.1.21:5403
       INFO  http server listening  addr=:8080

6. スマホ / タブレットのブラウザで、ゲートウェイ PC の IP を開きます:
       http://<このPCのIP>:8080/
   PC の IP がわからないときは、同じ黒いウィンドウのログから探すか、
   コマンドプロンプトで ipconfig を実行して IPv4 アドレスを確認します。

7. ブラウザで「設定」を開き、自分のトランスポンダー番号を入れて保存します。
   読み上げを使う場合は、表示されるロックボタン(🔊)を一度タップして
   音声を有効化してください(iPhone Safari の仕様で、初回タップが必須です)。


■ 終了手順
-----------

- 黒いウィンドウで Ctrl + C を押すか、ウィンドウを閉じます。
  ログ・記録ファイルが安全にフラッシュされてから終了します。
- USB を抜く前に必ず終了してください。
- 起動中に USB を抜いてしまっても、ゲートウェイ自体は動き続けますが、
  ログと録画ファイルは書けなくなります(警告ログが出ます)。


■ うまくいかないとき
---------------------

▸ gateway.exe が起動しない
  - SmartScreen で「詳細情報 → 実行」を押せていますか?
  - 一度起動を許可すれば、次回以降は警告は出ません。

▸ TCP に接続できない / ログに「dial: ...」が連続して出る
  - config.json の upstream.host / port が AMB の値と一致していますか?
  - PC と AMB が同じ LAN(同じネットワーク)にいますか?
  - 念のため一度終了して、AMB を再起動してから再試行してください。

▸ スマホからブラウザで開けない
  - URL は「http://<このPCのIP>:8080/」 ですか?
  - スマホとゲートウェイ PC が同じ WiFi につながっていますか?
  - 起動手順 §3 の Firewall 許可ルールを追加しましたか?
    パブリックネットワークではダイアログが出ずに無言でブロックされる
    ため、ルールを足さないとスマホから接続できません。
  - 別の PC にコピーした場合、新しい PC で §3 を再実行する必要があります
    (ドライブレターやフォルダが変わるため、既存ルールは効きません)。

▸ 音声が鳴らない
  - 設定画面で「読み上げを有効にする」がオンになっていますか?
  - 初回はロックボタン(🔊)を必ず一度タップしてください(iPhone Safari の制約)。
  - スマホのサイレントスイッチ・音量を確認してください。

▸ 動作はしているが LAN で他の人にも見えてしまうのが心配
  - 本アプリは LAN 専用です。Wi-Fi のパスワードでアクセスを制限してください。
  - インターネットに公開する設計にはなっていません(認証・暗号化なし)。


■ ファイル構成
---------------

このフォルダにある(または起動時に自動生成される)もの:

  gateway.exe ............ 本体(ダブルクリックで起動)
  config.example.json .... 設定のひな型(初回コピー元、編集しない)
  config.json ............ あなたの設定(初回起動時に自動作成)
  setup-firewall.bat ..... Firewall 許可ルール追加用(初回起動前にダブルクリック)
  README.txt ............. このファイル
  logs/ .................. ゲートウェイのログ(自動作成、自動ローテーション)
  records/ ............... --record オプション利用時の録画(通常は空)
